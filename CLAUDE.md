# CLAUDE.md — Architecture & Instructions

## Project Purpose
Interview Quiz Learning Space — a modular quiz platform for JavaScript, DSA, and React interview prep.

## Approach
Topic-based modular design. Each topic (JavaScript, DSA, React) has its own quiz set, scoring, and progress tracking. Redux Toolkit slices keep state predictable and testable.

## Architecture

```
React Components → Redux Store → Service Layer → External APIs
```

- **Frontend (React 18)** — renders UI, dispatches Redux actions
- **Redux Toolkit** — manages global state (quiz, results, user)
- **Service Layer** — abstracts Firestore and Groq calls; components never call APIs directly
- **Firebase** — Auth + Firestore for persistence
- **Groq API** — free AI question generation (llama-3.1-8b-instant)

## Component Map
| Component | Responsibility |
|-----------|---------------|
| `TopicSelector` | Choose JavaScript, DSA, or React topic |
| `QuizList` | 3-section browse page: Default Practice / AI Generate / Create Quiz |
| `QuizPlayer` | Step through questions, capture answers |
| `ResultPage` | Show final score + per-question explanations |
| `Dashboard` | Progress stats + all 3 quiz sections + recent activity |
| `QuizForm` | Create custom quiz questions |

## Quiz Source Types
Every Firestore quiz carries a `source` field:
| Value | Meaning |
|-------|---------|
| `'ai'` | Generated by Groq — shown in AI Generate section |
| `'user'` | Created manually via QuizForm — shown in Create Quiz section |

Static quizzes (`src/data/staticQuizzes.js`) use prefixed IDs (`static-javascript`, `static-dsa`, `static-react`) and never touch Firestore.

## Redux Slices
| Slice | Key State |
|-------|-----------|
| `quizSlice` | `quizzes` (topic), `globalQuizzes` (static), `firestoreQuizzes` (all Firestore), `currentQuiz` |
| `resultSlice` | `history`, `latestResult` |
| `userSlice` | `currentUser`, `authReady` (blocks render until Firebase confirms session) |

## Service Layer
| Service | Responsibility |
|---------|---------------|
| `quizService.js` | Firestore CRUD; `getQuizById` handles `static-*` IDs locally |
| `authService.js` | Firebase Auth (email + Google) |
| `aiService.js` | Groq generation with recycling logic (see below) |

## AI Generation Logic (`aiService.generateQuestions`)
Each call produces exactly **10 questions**:
1. Fetch all questions from prior AI quizzes for this topic from Firestore
2. Randomly recycle **3–5** from the existing pool (spaced repetition)
3. Call Groq for the remaining **5–7 new** questions, passing existing question texts as exclusions
4. Shuffle recycled + new into one set

First generation (no prior AI quizzes): all 10 are brand new.

## Auth Flow
- `authReady` flag in `userSlice` — app renders nothing until Firebase's `onAuthStateChanged` fires once, preventing route flicker on page load
- `GuestRoute` — redirects already-logged-in users away from `/auth`
- `ProtectedRoute` — redirects unauthenticated users to `/auth`
- Logout dispatches `resetQuiz` + `clearHistory` before setting user to null

## Key Rules for Claude
1. **Never call APIs directly in components** — always go through the service layer.
2. **Scoring logic lives in `utils/scoring.js`** — not in components or reducers.
3. **Use Redux Toolkit** (`createSlice`, `createAsyncThunk`) — no raw Redux.
4. **No caching AI responses** — each generation is intentionally fresh with recycled questions.
5. **No API keys in source** — always read from `process.env.REACT_APP_*`.
6. **Error boundaries** — wrap async-heavy components (`QuizPlayer`, `Dashboard`).
